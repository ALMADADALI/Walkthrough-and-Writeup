# OverTheWire: Bandit — Level 16 → Level 17

> Skills: Port scanning, nmap, openssl s_client, SSH key files
>
> Difficulty: Intermediate
>
> [Bandit Level 17](https://overthewire.org/wargames/bandit/bandit17.html)

## Login

```bash
ssh bandit16@bandit.labs.overthewire.org -p 2220
```

Password: *obtain from previous level*

## Task

> The credentials for the next level can be retrieved by submitting the password of the current level to a port on localhost in the range 31000 to 32000. First find out which of these ports have a server listening on them. Then find out which of those speak SSL and which don't. There is only 1 server that will give the next credentials, the others will simply send back to you whatever you send to it.

## Theory

| Concept | Explanation |
|---|---|
| Port | A numbered endpoint (0-65535) on a host where a service can listen for connections |
| `nmap -p <range>` | Scans only the specified port range instead of nmap's default top 1000 ports |
| `nmap -sV` | Adds service/version detection on top of the port scan, identifying what's running and whether it's SSL |
| `nmap -p-` | Scans all 65535 ports — you won't need this here but good to know |
| `nmap -A` | Runs an aggressive all-in-one scan (OS detection, version detection, scripts, traceroute) — you won't need this here but good to know |
| `openssl s_client -connect <host>:<port>` | Connects to an SSL/TLS service, as covered in Level 16 |

## Solution

### Step 1: Scan ports 31000-32000 for open services with version detection

The task gives a 1000-port range to search. `-sV` tells nmap to identify what's actually running on each open port, including whether it speaks SSL.

```bash
bandit16@bandit:~$ nmap -sV localhost -p 31000-32000
```

> What to notice: Five ports are open, but only two — 31518 and 31790 — run over SSL. Port 31518 is identified as `ssl/echo`, meaning it just echoes back whatever you send. Port 31790 is `ssl/unknown`, which is the most promising candidate since its service isn't immediately recognizable as a simple echo.

> What to notice: Instead of a plain text password, the server returns a full RSA private key. This is the credential for bandit17 — you'll use it with `ssh -i` instead of a password, the same way you used `sshkey.private` to log into bandit14.

### Step 3: Save the private key to a file and restrict its permissions

Copy the entire key block (including the `-----BEGIN-----` and `-----END-----` lines) into a file, then lock down its permissions — SSH refuses to use private key files that are readable by other users.

```bash
bandit16@bandit:~$ nano sshkey17.private
bandit16@bandit:~$ chmod 600 sshkey17.private
```

```
(no output provided — these commands produce no terminal output on success; nano opens an editor where you paste the key and save, and chmod silently changes file permissions)
```

> What to notice: No output is the expected and correct result here. If `chmod` is skipped, the next SSH login will fail with a "permissions are too open" error rather than connecting.

## Key Takeaways

Combine `nmap -sV` with a port range to quickly narrow down candidates, then filter by SSL and discard known "echo" services. Credentials aren't always passwords — here the reward is a private key, which must be saved with `chmod 600` before SSH will accept it.

## Common Mistakes

| Mistake | What Happens | Fix |
|---|---|---|
| Trying port 31518 instead of 31790 | Server just echoes back whatever you send, no real password | Check the nmap version column — `ssl/echo` is a dead end, `ssl/unknown` is the target |
| Skipping `chmod 600` on the saved key file | SSH refuses the key with a "permissions too open" or "bad permissions" error | Run `chmod 600 sshkey17.private` before using it with `ssh -i` |
| Forgetting to copy the full key including BEGIN/END lines | SSH fails to parse the key file | Copy the entire block exactly as returned, including both header and footer lines |
| Not closing the openssl connection after receiving the key | Terminal appears stuck in the SSL session | Press Ctrl+C or Ctrl+D to exit, then proceed to save the key |
